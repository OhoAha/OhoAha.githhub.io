SECURE SOFTWARE DEVELOPMENT LIFECYCLE


INTRODUCTION
This site is about educating on SSDLC by discussing each security measures needed in the normal SDLC. There are 6 major components that are highlighted to be important. We describe useful matters in each components which the red boxes in the diagram shown. For each SSDLC step, we provide definition, related cases, solution and tools.


Conventional SDLC
Software Development Life Cycle is a framework that defines the development process used by organizations to build an application from initial until the real product can be used. It is a continuous process starting from the decision to launch the project and ends at the moment it is released to the targeted market where the products need to be maintained. Conventional SDLC consists of six phases which are requirement and use case, architecture and design, test plan, code, conduct test and go live.
Secure SDLC
Observation of security vulnerabilities from previous studies justifies that the conventional SDLC is not enough to secure a software from the attack of security threats. It is essential to incorporate Secure SDLC to ensure the systemâ€™s functionality and curb security bugs in software applications. An improved secure SDLC (SSDLC) which consists of security requirements, threat modelling, risk analysis, risk-based security test, code security review, vulnerability assessment and penetration testing, and security monitoring/surveillance is introduced.
